## 支持的版本

OreoDeck 的最新发布版本会获得安全修复。

| 版本 | 是否支持 |
| --- | --- |
| 0.1.x | 是 |
| 更早或尚未发布的构建 | 否 |

## 报告漏洞

请勿在公开 issue 中披露漏洞、凭据、账户信息或漏洞利用细节。

请通过 `OreoSolutions/oreodeck` 的 GitHub 私有漏洞报告或私有安全公告功能提交，并包含：

- 受影响的 OreoDeck 和 macOS 版本。
- 复现步骤或最小概念验证。
- 预期和实际观察到的安全影响。
- 是否涉及配置档案数据、API 密钥、OAuth 状态、Shell 集成、符号链接或终端命令执行。

维护者将在审阅后确认完整报告，进行私下调查，并在可行时与报告者协调披露。我们不保证具体的响应或修复时限。

## 范围

高优先级领域包括凭据泄露、绕过配置档案隔离、路径遍历、不安全的符号链接处理、终端命令注入、不安全的 Keychain 访问，以及卸载操作删除 OreoDeck 管理范围之外的数据。

Claude Code、终端应用、macOS 和其他第三方依赖拥有各自的安全政策。仅影响这些项目的漏洞也应向相应上游报告。

> 如解释存在差异，以英文政策为准。
